# 🤝 Правила внесення змін до проєкту (Contributing Guide)

Дякуємо, що ви хочете зробити свій внесок у розвиток **BadCalculator**! Дотримання цих правил допомагає підтримувати стабільність проєкту та чистоту історії версій.

---

## 🚀 З чого почати?

1. **Створення робочої гілки:**
   Всі виправлення помилок або додавання функціоналу мають відбуватися в окремих гілках. Створюйте гілку від `main` зі зрозумілим префіксом:
   * `fix/назва-багу` — для виправлення помилок
   * `feat/назва-функції` — для додавання нового коду
   * `docs/назва-файлу` — для покращення документації

2. **Format повідомлень комітів:**
   Ми використовуємо чітку структуру для повідомлень комітів:
   * `fix: опис виправлення` (наприклад, `fix: додано обробку ділення на нуль`)
   * `feat: опис нововведення` (наприклад, `feat: додано unit-тести для кнопок`)
   * `docs: зміни в документації`

3. **Автоматичне закриття тасок (Issues):**
   При зв'язуванні виправлення із баг-репортом, завжди додавайте в опис вашого коміту або Pull Request ключові слова, наприклад: `closes #2`.

---

## 🧪 Вимоги до тестування
* Перед відправкою коду (Push) обов'язково запустіть локальні модульні тести: `pytest test_pytest.py`.
* Якщо ви додали нову функцію або змінили логіку обчислень, напишіть для неї відповідний тест-кейс. Всі тести мають завершуватися зі статусом **PASSED**.

---

## 📬 Створення Pull Request (PR)
1. Відправте локальну гілку на GitHub.
2. Створіть запит на злиття через інтерфейс GitHub (**New Pull Request**).
3. Перевірте, щоб ваш код не мав конфліктів із гілкою `main` та успішно проходив автоматичні перевірки.
